# POS Transaction Engine

Production-grade POS backend demonstrating five reliability patterns
that solve real problems at scale.

When two cashiers simultaneously bill the last item — who wins?
When a network drops mid-payment — does the customer get charged twice?
When a server crashes — are the audit logs lost?

This system answers all three with working, production-grade code.

## What this is

A demo project where you can create orders, see inventory manipulation
after each order. Data consistency, audit logging, and concurrency
control are the core focus.

## Why I built it

To demonstrate production reliability patterns through working code —
not slides, not theory. A live demo you can break and reset.
